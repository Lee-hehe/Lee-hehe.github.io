Linux教程——CentOS 7

Linux常用命令，软件安装以及项目部署

一、引言

1.1 开发环境

平时大家大多是在Windows或者Mac操作系统下去编写代码进行开发，在开发环境中安装大量的软件，这样会导致环境的稳定性和安全性降低

1.2 生产环境

是将程序运行在此环境中，供用户去使用。这个环境是有专业的人员去维护，一般 人是没有权限去操作生产环境的

1.3 测试环境

一般克隆一份生产环境，会将开发环境中的程序部署到测试环境中，这个环境的主要目的是去程序进程检测，收集程序中的各种问题，并交给开发人员进行修改

1.4 操作系统的选择

生产环境中，常用的操作系统有Windows 2003 service、Linux、Unix等等，Linux操作系统， 在生产环境中占据了大量的市场份额，Linux主要以稳定、可靠、免费的特点成为全球使用最多的服务器操作系统。

Linux操作系统现在已经成为后台开发人员必备的技能，并且后期学习的各种知识都会涉及到Linux操作系统。

二、Linux介绍

2.1 Linux介绍

在Linux操作系统出现之前，还有一个操作系统叫做Minix，Minix操作系统是由Andrew的大学教授研发出来的，当时大学教授是为了给学生上课，买了一套Unix操作系统，参考Unix自己写了一个操作系统，并且命名为Minix。同时将Minix开源， 供学校内部的研究和教学，到了2000年，Andrew将Minix操作系统完全对外开源。

Minix由于完全对外开源之后，在互联网上迅速的传播，但是大家在使用时，发展Minix不是很完美，内部存在各种各样问题。用户将问题解决后，编写了一个补丁，将补丁以邮件的方式发给Andrew，但是Andrew教授最初的目的只是为了教学和研究。

与此同时，一位芬兰的大学生出现了，叫Linus，在Minix操作系统基础上，自己添加了一些补丁和插件，并将其命名为Linux操作系统，并且完全对外开源，而且开始维护Linux操作系统。Git也是Linus研发的。

2.2 Linux的版本

Linux的版本分为两种，一种是Linus团队正在维护的开源内核版本。另一种是一 些厂商基于内核版本封装的发行版本。

  • Linux的内核版本官网
  • Linux的发行版本，这里演示的发行版本是CentOS

Linux的常见发行版本

2.3 Linux和Windows的区别

  • Linux是严格区分大小写的，Windows无所谓。
  • Linux中一 切皆是文件。
  • Linux中文件是没有后缀的， 但是他有一 些约定俗成的后缀。
  • Windows下的软件一般是无法直接运行的Linux中。

四、Linux的目录结构

Windows的目录结构是带有盘符的。D：、E：、C：，而Linux中是没有盘符的

在Xterm中输入ls /查看Linux的顶级目录。

Linux的常用目录……

  • root：该目录为系统管理员HOME目录
  • bin：这个目录下放着经常使用的命令
  • boot：这里存放的是启动Linux时的一些核心文件
  • etc：存放系统管理所需要的配置文件和子目录
  • home：普通用户的HOME目录
  • usr：默认安装软件的目录，类似Windows中的Program Files目录
  • opt：是主机额外安装软件拜访的目录

五、Linux的常用命令

5.1 基本命令

命令的模式：命令 [选项] 操作的文件或路径

  • ifconfig：查看IP地址等信息
  • su [xx]（switch user）：更换用户，如果直接写su表示更换成root管理员账号
  • pwd（print working directory）：打印当前所在路径
  • cd xx（change directory）：进入文件夹
  • cd ..：返回上一级文件夹
  • cd /：返回根路径
  • ll：打印当前文件夹下的所有子文件和子文件夹的详细信息
  • ls：打印当前文件夹下的所有子文件和子文件夹的名称
  • clear：清空命令行

su sashixin

注意：从root切换成其他用户不需要输入密码，但从其他用户切换到root账户需要密码

cd xx

  • mkdir xx：在当前路径下创建一个文件夹

  • mv：移动文件/移动并重命名文件

    • 被移动的文件 /移动到的路径/重命名后的文件

  • rm：移除文件

    • -r：移除文件夹，同时删除该文件夹下的所有文件和子文件夹
    • -f：强制删除文件
    • -rf：强制删除文件夹

  • cp（copy）：复制文件

mkdir xx

mv 被移动的文件 /移动到的路径/重命名后的文件

5.3 文件内容的操作

5.3.1 创建或者编辑文件

  • touch fileName：创建一个文件
  • vim xxx：创建一个空白文件，进入编辑页面，如果文件存在则编辑页面（若未安装图形化界面，则命令只有vi）

5.3.2 文件三种模式

  • 一般模式：复制一行，粘贴
  • 编辑模式：i|a|o对文件进行修改，esc退出
  • 底行模式：显示行号，取消行号，保存退出，不保存退出

  • :set nonumber：关闭文件行数

:set number        #显示行号

:n        #定位到第n行
gg        #移动到文件第一行
G            #移动到最后一行

5.3.4 撤销命令（在一般模式下运行）

5.3.5 行复制（在一般模式下运行）

5.3.6 删除命令（在一般模式下运行）

  • dd：删除光标所在行

5.3.7 退出文件编辑

  • :wq：保存并退出
  • :q!：强制退出不保存

5.3.8 仅查看文件

  • cat xxx：直接查看文件所有内容，
    • -n：显示行号
  • tail -f xxx：可以用来查看日志
  • more xxx：查看大文件时，可以一页一页的向下翻，按space向下翻页，退出时按q
  • nl xxx：从最后一行开始展示（显示行号）
    • -ba：表示不论是否为空行，也同样列出行号（类似cat -n）

5.6 权限的操作

5.6.1 理解文件描述符

d rwx rwx rwx

文件类型区（d）

  • d：表示这是一个文件夹
  • -：表示这是一个文件

用户权限区（u）

  • 表示当前用户所拥有的权限

组权限区（g）

  • 表示同组用户所拥有的权限

其他权限区（o）

  • 表示其他非同组用户所拥有的权限

5.6.2 理解rwx

  • r：表示可读权限
  • w：表示可写权限
  • x：表示可执行权限
  • -：表示无

5.6.3 修改文件权限

  • 字符表示法
    • chmod u|g|o + r|w|x 文件名或文件夹`
      • u`：所属用户权限
      • g：同组权限
      • o：非同组用户权限
      • +：添加权限
      • -：移除权限
      • =：覆盖当前权限
chmod u-x,g+r,o=rwx ./a.txt        
#为当前用户移除可执行权限，为同组用户台南佳可读权限，为非同组用户覆盖权限为可读可写可执行

  • 数字表示法
    • chmod 700 文件名或者文件夹`
      • r`：4
      • w：2
      • x：1
chmod 700 文件名或者文件夹


chmod 666 文件名或者文件夹

  • -R
    • chmod -R 777 文件夹：同时授权文件夹下的所有子文件
chmod 777 文件夹/                #授权文件夹


chmod -R 777 文件夹/        #同时授权文件夹下的所有子文件

    • pid：进程的id
    • -9：表示强制终止
kill -9 pid        #杀死进程

8.1 什么是Linux服务

8.2 服务的操作命令

    • start：开启，启动
    • stop：关闭，停止
    • restart：重启
    • reload：重装
    • status：查看状态
systemct1 status firewalld        #查看防火墙状态

10.5 配置环境变量

Linux提供了两种环境变量的文件

  • 第一个是用户级别的环境变量，存放在：-/.bashre
  • 第二个是系统级别的环境变量，存放在：/etc/profile

修改哪个文件都可以，毕竟虚拟机就我们自己使用

vim /etc/profile        #修改配置文件

在环境变量中添加如下内容

export JAVA_HOME=/use/local/jdk        #定义JAVA_HOME
export PATH=$JAVA_HOME/bin:$PATH    #把JAVA_HOME添加到PATH

  • $：代表使用环境变量
  • :：代表Windows中的;
  • $PATH：代表使用之前的path

esc，:wq：保存退出

重新加载环境变量文件并查看

source /etc/profile    #重新加载环境变量文件
java -version                #最终测试

10.6 安装Tomcat

tar -zxf Tomcat压缩包.tar.gz        #解压
rm -rf ./*.gz                                     #删除当前目录下所有的.gz压缩包
cd Tomcat目录                                    #进入Tomcat目录


cd bin/                #进入bin目录下
./startup.sh    #运行程序


ps -ef | grep 服务                     #查看服务进程
netstat -naop | grep 端口号    #查看端口正在被谁占用


kill -9 进程号                            #杀死进程
curl http://localhost:8080    #访问本地8080端口


tail -f catalina.out        #查看日志

此时服务已启动，但是只能在虚拟机中访问，Windows中无法访问，因为存在防火墙

有两种方式可以访问，一种是关闭防火墙，但会导致任何人都可以访问

另一种是配置开放的端口号（如果要配置开放某个端口号，必须在ssh.xml中开放）

vim /usr/lib/firewalld/services/ssh.xml        #进入ssh.xml配置端口号

在ssh.xml文件中添加下面的内容

<!--配置开发8080端口号-->
<port protocol="top" port="8080"/>

esc，:wq：保存退出

systemctl restart firewalld        #重启防火墙

注意：在Windows中访问tomcat要写虚拟机的IP:8080
