从零开始的个人博客搭建

发表于2020-05-21更新于2020-05-23字数统计2.7k

一、如何部署环境

1.安装Node.js，npm

a. 进入Node.js官网

下载LTS长期支持版，一直下一步即可。

安装完成会有两个组件：“Node.js”和“npm包管理器”

b. 打开cmd（Windows）/iTerm（IOS）（ios需切换到root用户：sudo su）

  • 查看node版本：node -v

  • 查看npm包管理器的版本：npm -v

2.安装镜像源cnpm

安装需要借助npm包管理器来安装，但由于国内镜像源太慢，一般在使用之前利用npm来安装cnpm

  • 全局安装指向淘宝的镜像源cnpm：npm install -g cnpm --registry=https://registry.npm.taobao.org

  • 查看cnpm版本：cnpm -v

3.安装hexo博客框架

  • 用cnpm全局安装hexo博客框架：cnpm install -g hexo-cli

  • 查看hexo版本：hexo -v

此时hexo就已经安装完了

二、如何使用hexo搭建博客

  • 查看当前路径：Windows：chdir，IOS：pwdrhcl

1.新建一个文件夹

  • 新建空的文件夹“blog”：mkdir blog

  • 进入“blog”文件夹：cd blog/

2.使用hexo生成博客

  • 使用hexo生成博客：Windows：hexo init，IOS：sudo hexo init

查看当前目录下的文件：

  • Windows：dir，IOS：ls -l

这些文件都是Hexo生成的博客框架的基本内容

此时已经安装完成了，我们来启动博客

  • 启动博客：hexo s

（hexo s不能用的话先输入npm install hexo-server）

提示在本地localhost的4000端口启动

  • 打开浏览器，访问localhost4000端口：http://localhost:4000

可以看到博客已经创建好了，而且默认创建了一篇标题为“Hello World”的文章，里面简介了hexo的使用方法

3.新建一篇文章后编辑

  • 先断开掉：Ctrl + c

  • 创建一篇文章：hexo n "此处为博客文章名称"

  • 进入文章所在文件夹：cd source/_posts/

  • 查看该文件夹下的所有文件：Windows：dir，IOS：ls -l

可直接用markdown编辑

  • IOS也可使用vim编辑文章：vim 文章名称.md

基于markdown的格式写

  • 写完保存退出：esc,wq,回车

4.重新生成再预览

  • 返回两层回到blog文件夹下：cd ../..

  • 先清理一下：hexo clean

  • 再生成一下：hexo g

  • 启动预览博客：hexo s

重新进入4000端口：

此时博客的搭建已经完成

三、如何将个人博客部署到远端

博客搭建完成不可能只在本地用，需要部署到远端去

这里推荐一个免费的部署方式：通过部署到GitHub的地址来访问博客

接下来演示如何将博客部署到GitHub上去

关于GitHub账号的创建可以参考—>xingkaifan：创建我的GitHub账号及使用

1.在GitHub上新建一个仓库

  • 首先断开：Ctrl + c

登录自己的GitHub

然后新建一个仓库

用户部署个人博客的GitHub仓库的命名必须复合特定要求！

1处必须是这个格式：“你的昵称”.github.io，后续在浏览器中输入这个网址就能访问自己的博客

在2处填写一个描述，例如“我的博客”

选择3，点击4，即可创建完成

2.安装Git部署插件

  • 然后在blog目录下安装Git部署插件：cnpm install --save hexo-deployer-git

3.设置“_config.yml”文件

安装完成之后需要设置“_config.yml”文件

打开“_config.yml”文件（我这里直接用的IDEA打开的）

（IOS可以输入vim _config.yml直接修改）

找到“# Deployment”（一般在最下面），进行一些配置

1：直接在引号中加上“git”

2：先写“repo：”，然后去刚刚创建好仓库的页面复制网址，粘贴过来

3：先写“branch：”， 然后写“master”

IDEA直接退出即可 (IOS需要esc，wq，回车)

4.部署到远端

  • 接下来部署到远端：hexo d

注意！1处填写自己github的账户，2处填写自己账户的密码 （输入密码的时候不会显示出来！！！）

当推送成功之后，可以刷新一下仓库，可以看到会出现很多新的文件

这时我们复制这个网址的后面一段（方框内）

然后直接打开复制的网址，此时就已经成功地部署到远端了

四、如何更换博客主题

1.下载目标主题

演示主题的网址：github.com/litten/hexo-theme-yilia

  • 在blog文件夹下克隆目标主题：git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia

下载完成之后配到自己的博客里去

2.修改“_config.yml”文件

按照刚才的方法打开“_config.yml”文件，找到“theme”

将方框内的名称改为想换的主题的名称

然后保存退出

3.重新执行“clean，g，s，d”

  • 清除hexo clean

  • 生成hexo g

  • 执行hexo s

此时再刷新4000端口，会发现主题已经切换完毕

  • 然后Ctrl + c退出

  • 推到远端：hexo d

刷新自己的网址（每次更改主题后不会立马生效，需要等一会），刷新成功后主题就已更换完毕！

五、使用SSH密钥连接GitHub

下面是使用Git Bush来运行而不是用cmd的方法

1.下载Git

a. 进入Git官网

选择自己的电脑型号的位数，点击下载

b. 按照图示进行安装

c. 安装完成后检验是否安装成功

回到电脑桌面，鼠标右击，如果看到有两个git单词则安装成功

d. 接下来需要配置Git的一些基本信息

右键点击Git Bash Here

  • 设置用户名：git config --global user.name '你在GitHub上注册的用户名'；
  • 设置用户邮箱：git config --global user.email '注册时候的邮箱'；
  • 配置完成之后，使用git config --list命令看是否配置成功。

e. 若要完成之前的命令，直接在新建的blog文件夹下，鼠标右键点击Git Bash Here，然后依然按照之前的步骤执行

五、小结

先来回顾一下过程：

1.首先是环境准备，安装node.js、git，接下来安装hexo；

2.利用hexo init命令来初始化并编辑一个博客；

3.将个人博客部署到GitHub上去；

4.最后更换主题等其他。

这里建议先把博客搭建起来，然后再考虑主题、内容、页面美观等其他方面的问题，不要着急，一步步来。

安装过程中难免会出现错误，这时不要怕，大不了重头再来，人不能缺少重头再来的勇气！
